# RestaRatio

RestaRatio — это cooking simulator prototype (overcooked-like), разработанный на Unity с фокусом на архитектуру игровых систем, управление состояниями и взаимодействие игровых объектов.

Проект используется как sandbox для разработки масштабируемых игровых систем и архитектурных решений.

---

# Цели

Основная цель проекта — реализация чистой и модульной архитектуры, которая позволяет:

* безопасно расширять игровые системы
* управлять зависимостями
* реализовывать event-driven взаимодействие
* разделять ответственность между компонентами

---

# Архитектура

Проект использует следующие архитектурные подходы:

* Dependency Injection (Zenject)
* Reactive programming (UniRx)
* Async operations (UniTask)
* Addressables для управления ассетами

Архитектура построена таким образом, чтобы игровые системы были слабо связаны и легко расширяемы.

---

# Системы

## Заказы

Реализована система заказов, включающая:

* создание заказов
* управление жизненным циклом заказа
* взаимодействие с игроком
* обновление UI

Система использует реактивный подход для обработки событий.

---

## Взаимодействия

Реализована система взаимодействия игрока с объектами:

* выбор объектов
* взаимодействие с игровыми станциями
* передача данных между системами

---

## Состояния

Реализовано управление состояниями игровых систем:

* игровые состояния
* состояние заказов
* состояние взаимодействия

Используется event-driven подход.

---

## Внедрение зависимостей

Zenject используется для:

* управления зависимостями
* уменьшения связанности систем
* улучшения архитектуры проекта

---

## Асинхронная логика

UniTask используется для:

* асинхронных игровых операций
* управления игровыми последовательностями
