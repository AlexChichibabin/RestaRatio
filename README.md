# RestaRatio (Overcooked-like)

**RestaRatio** — это аркадный тайкун в духе *Overcooked 2*, где игрок управляет кухней, готовит блюда, обслуживает заказы и оптимизирует рабочие процессы, чтобы успевать в условиях ограниченного времени.

Проект создаётся как pet-project с фокусом на:
- геймплейную логику  
- архитектуру (Zenject, Addressables)
- оптимизацию  
- реактивный подход (UniRx)
- асинхронность (UniTask)

---

## Геймплей

- управление персонажем на кухне  
- выбор персонажа перед матчем (персонажи влияют на геймплей)  
- взаимодействие со станциями (резка, жарка, сборка блюд)  
- система заказов с таймерами  
- уникальные усиления во время матча  
- начисление очков довольства за скорость и качество готовки 
- рост сложности по мере прогресса (возможно)

---

## Персонажи и усиления

### Персонажи
Каждый персонаж имеет уникальные особенности, влияющие на геймплей, например:
- скорость передвижения  
- скорость приготовления  
- бонусы к очкам  
- особые пассивные эффекты  

---

### Усиления
Во время матча игрок может получать усиления:
- ускорение действий  
- мгновенное приготовление  
- увеличение награды за заказы  
- замедление таймера заказов
- какие-то еще 

Усиления накладываются через систему эффектов и действуют до конца натча как правило.

## Архитектура и ключевые системы

### Order System на UniRx

- каждый заказ — отдельная модель с состоянием и таймером  
- используется `ReactiveProperty` для:
  - времени до истечения  
  - состояния заказа (Active / Completed / Expired)  
- генерация заказов через `Observable.Timer`  
- UI подписывается на изменения без `Update()`  
- обработка завершения и истечения заказов через реактивные стримы  

---

### Player Interaction

- ввод игрока как поток событий  
- InDev: предотвращение спама действий (`ThrottleFirst`) 
- InDev: комбинирование ввода и контекста (`WithLatestFrom`)  

---

### UI Binding

- HUD обновляется через подписки  
- данные → UI, без жёсткой связи  
- список заказов формируется динамически
- InDev: анимации и эффекты запускаются через события  

---

### InDev: Character System

- выбор персонажа перед матчем  
- параметры персонажей (скорость, бонусы, эффекты) вынесены в данные  
- применение бонусов через реактивные свойства  
- влияние персонажа на:
  - скорость готовки  
  - таймеры заказов  
  - награды за выполнение  

---

### InDev: Power-up System

- усиления представлены как эффекты, изменяюшие геймплей или баланс персонажа
- эффекты применяются в определенных точках игры
- возможность комбинировать эффекты  
- события активации / завершения усилений
